## 项目概览

本项目在 **ALINX AX7020 (Xilinx Zynq-7020)** 平台上构建了一套端到端的实时运动目标检测与跟踪系统：

> **OV5640 摄像头采集 → DDR3 帧缓冲 → 帧间差分 → 形态学滤波 → 外接矩形 → VGA 实时显示**

整条算法链路全部在 PL 端用 Verilog 流水实现，无需 PS 介入即可完成从采集到显示的闭环。设计目标是用最经典的 **帧间差分法（Inter-frame Difference）** 在硬件上跑通完整的视频流水线，验证算法在 FPGA 上的可流水化与实时性。

## 系统架构

| 层级         | 实现                                       |
| ------------ | ------------------------------------------ |
| **图像采集** | OV5640 CMOS 摄像头，I²C 配置 + DVP 接口    |
| **视频缓存** | DDR3 帧缓冲，乒乓存储相邻两帧              |
| **核心算法** | 帧间差分 → 二值化 → 形态学 → 轮廓提取      |
| **目标定位** | 实时计算运动区域外接矩形                   |
| **显示输出** | VGA 输出实时画面 + 跟踪框叠加              |
| **平台**     | Zynq-7020（PL + PS 协同，算法跑在 PL）     |

## 算法流水线

整条流水线在 FPGA 内以**像素时钟**为节拍逐帧推进，端到端低延迟：

### 1. 灰度化

将 RGB 转换为单通道灰度，简化后续运算：

```
Gray = 0.299·R + 0.587·G + 0.114·B
```

### 2. 平滑滤波

3×3 均值滤波抑制 CMOS 噪声，避免帧差结果出现"闪烁噪点"。

### 3. 帧间差分

对相邻两帧灰度图取像素差的绝对值：

```
D(x,y) = | I_t(x,y) − I_{t−1}(x,y) |
```

静止背景被消去，只剩下运动区域的变化。

### 4. 二值化

阈值二值化得到运动掩膜：

```
M(x,y) = 1, if D(x,y) > T
       = 0, otherwise
```

阈值 T 通过实验调参得到，平衡漏检与误检。

### 5. 形态学滤波

- **腐蚀**：去除离散的孤立噪声点
- **膨胀**：填补目标内部的小孔与断裂
- 组合即"开运算 + 闭运算"，得到连通性更好的目标区域

### 6. 外接矩形与跟踪框

- 行/列扫描得到运动目标的最小外接矩形
- 将矩形坐标叠加到原始视频流上，VGA 输出带跟踪框的实时画面

## 工程实现

- **完整视频流水线自研**：从摄像头驱动、I²C 配置、DDR3 帧缓冲到 VGA 显示全部从零搭建
- **PL 端纯硬件流水**：算法核心用 Verilog 实现，每个像素周期推进一级流水，满足实时性
- **模块化设计**：摄像头驱动、帧差核心、形态学单元、外接矩形、VGA 输出各自独立，便于复用与调参
- **平台**：ALINX AX7020 (Zynq-7020 + DDR3 + OV5640 + VGA)

## 局限与展望

- 帧差法对**光照剧变**与**慢速目标**敏感：剧变会引入大面积伪运动，慢速目标差分信号弱
- 当前外接矩形跟踪较为粗糙，后续可引入 **更精确的轮廓提取**（如 Sobel + 连通域）或 **背景建模**（混合高斯）以提高精度
- 可扩展为多目标跟踪并结合卡尔曼滤波实现轨迹预测
